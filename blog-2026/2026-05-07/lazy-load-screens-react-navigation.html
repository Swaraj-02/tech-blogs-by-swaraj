Lazy Load Screens with React Navigation to Shrink Bundle Size

📅 2026-05-07
Learn how to dynamically import screen components in React Navigation, reducing the initial bundle and speeding up app startup.
react navigation performance lazy loading

Insight

When a React Native app grows, every screen contributes to the initial JavaScript bundle, slowing cold starts. By lazy‑loading screens with React.lazy and Suspense, you defer code loading until the user actually navigates there. This pattern works seamlessly with React Navigation's createNativeStackNavigator and can cut bundle size by 10‑20% for medium‑sized apps.

Example

// navigation.tsx
import { createNativeStackNavigator } from '@react-navigation/native-stack';
import { Suspense } from 'react';
import Loading from './components/Loading';

const Stack = createNativeStackNavigator();

const HomeScreen = React.lazy(() => import('./screens/HomeScreen'));
const SettingsScreen = React.lazy(() => import('./screens/SettingsScreen'));

export default function AppNavigator() {
  return (
    <Stack.Navigator>
      <Stack.Screen name="Home">
        {() => (
          <Suspense fallback={<Loading />}> <HomeScreen /> </Suspense>
        )}
      </Stack.Screen>
      <Stack.Screen name="Settings">
        {() => (
          <Suspense fallback={<Loading />}> <SettingsScreen /> </Suspense>
        )}
      </Stack.Screen>
    </Stack.Navigator>
  );
}

Takeaway

Wrap each lazily loaded screen in a Suspense fallback and register it as a render prop (not a component) in the navigator. This keeps the navigation API intact while shaving off unused code from the initial load, delivering a snappier startup experience.
